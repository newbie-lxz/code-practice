# 分类规则说明

这次准确率问题主要出在最终标签规则太激进：很多弹幕本来模型原始判断是中性，但旧规则会按正/负概率强行二选一。

现在的策略是：

1. `哈哈`、`笑死`、`学到了`、`干货` 等明显正向弹幕，优先判正面。
2. `入土`、`坐牢`、`看不懂`、`学不会`、`太水` 等明显负向弹幕，优先判负面。
3. `打卡`、`报到`、`资料`、`笔记`、`倍速`、`数据`、`代码`、`函数`、`课程` 等说明性弹幕，优先保留中性。
4. `好`、`强`、`牛`、`水` 这种单字词容易误伤，已经从强情感词典规则里剔除。
5. 人工标注仍然优先级最高，会覆盖模型和规则。
